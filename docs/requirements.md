# 要件定義書: 独自絵文字言語 解析アプリ

## 1. プロダクト概要

紙に印刷された独自の絵文字言語をスマホカメラで撮影し、ローカルAIで即座に日本語に変換するPWAアプリ。

## 2. 最重要要件

- **速度最優先**: 撮影から解析完了まで体感で即座（100ms以内目標）
- **完全ローカル処理**: 通信不要、ブラウザ内でAI推論を完結
- **高精度**: 背景模様の微細な差異を正確に分類

## 3. 独自言語の仕様

### 3.1 文字構造

1つの「文字」は以下の要素で構成される：

| 要素 | 役割 | 詳細 |
|---|---|---|
| 前景の絵 | 母音（段）を決定 | あ段=恐竜, い段=竜, う段=頭蓋骨, え段=赤子, お段=サメ |
| 背景の模様 | 子音（行）を決定 | 模様の種類でカ行〜ワ行を区別。ア行は模様なし |
| サイコロ（右上） | 濁点 or 半濁点 | 濁点用サイコロ / 半濁点用サイコロの2種類 |
| サイズ | 通常 or 小文字 | 小さい = 小文字（ゃゅょっ等） |

### 3.2 例外

- 「ん」: カブの絵のみ（模様なし、他の文字と異なる構成）

### 3.3 分類対象一覧

- 母音（前景の絵）: 5種類 + 「ん」用1種類 = 6種類
- 子音（背景の模様）: ア行(なし), カ行, サ行, タ行, ナ行, ハ行, マ行, ヤ行, ラ行, ワ行 = 10種類
- 濁点/半濁点: なし, 濁点, 半濁点 = 3種類
- サイズ: 通常, 小文字 = 2種類

## 4. 機能要件

### 4.1 カメラ入力

- スマホカメラのリアルタイムプレビューを表示
- 撮影ボタンまたは自動検出でキャプチャ
- 画像の前処理（リサイズ、正規化等）をブラウザ内で実行

### 4.2 画像解析（ローカルAI）

- 撮影画像から個々の文字（アイコン）領域を検出・切り出し
- 各アイコンについて以下を分類:
  1. 前景の絵 → 母音を特定
  2. 背景の模様 → 子音を特定
  3. サイコロの有無・種類 → 濁点/半濁点を特定
  4. サイズ → 通常/小文字を特定
- 分類結果を組み合わせて日本語文字に変換

### 4.3 結果表示

- 解析結果（日本語テキスト）を画面に即座に表示
- 解析にかかった時間を表示（デバッグ/チューニング用）

## 5. 非機能要件

| 項目 | 目標 |
|---|---|
| 推論速度 | 1文字あたり5-20ms（ONNX Runtime Web） |
| 全体レスポンス | 撮影から結果表示まで100ms以内（推論部分） |
| オフライン対応 | 初回ロード後はオフラインで完全動作 |
| 対応端末 | iOS Safari / Android Chrome |
| 配布方法 | PWA（URL共有のみ、ストア不要） |

## 7. 開発フェーズ

### Phase 1: モデル訓練

1. 対応表から各音のアイコンを個別に拡大撮影（学習データ収集）
2. データ拡張（回転、照明変化、ノイズ等）
3. 画像分類モデルの訓練（PyTorch）
4. ONNX形式へ変換・精度検証

### Phase 2: Webアプリ開発

1. PWAの基盤構築（カメラ入力、UI）
2. ONNX Runtime Webの組み込み
3. 画像からアイコン領域の検出・切り出しロジック
4. 分類結果 → 日本語変換ロジック
5. 結果表示UI

### Phase 3: 精度・速度チューニング

1. 実際の紙での撮影テスト
2. 誤認識パターンの分析と改善
3. 速度最適化
